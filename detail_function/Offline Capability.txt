
# รายละเอียดฟังก์ชัน Offline-First Capability (ระบบทำงานต่อเนื่องแม้ออฟไลน์)

## 1. หลักการทำงานเบื้องต้น (Core Concept)
ระบบ Offline-First ถูกออกแบบภายใต้แนวคิด **"ทำงานได้ทุกที่ ไม่เกี่ยงสัญญาณ"** โดยแอปพลิเคชันจะให้ความสำคัญกับ **Local Storage (หน่วยความจำในเครื่อง)** เป็นอันดับแรก เมื่อผู้ใช้งานบันทึกข้อมูล ข้อมูลจะถูกเขียนลงในอุปกรณ์ทันทีเพื่อให้มั่นใจว่าข้อมูลไม่สูญหาย จากนั้นระบบ Background Service จะคอยตรวจสอบสัญญาณอินเทอร์เน็ตเพื่อส่งข้อมูลขึ้น Cloud (Google Sheets) ในภายหลังโดยอัตโนมัติ

## 2. กระบวนการทำงาน (Workflow)

### สถานะ: 🟢 ออนไลน์ (Online)
1.  **บันทึก:** ผู้ใช้งานกด "บันทึกข้อมูล"
2.  **ส่งข้อมูล:** แอปพลิเคชันส่ง API Request ไปยัง Google Sheets ทันที
3.  **ยืนยัน:** หากสำเร็จ จะแสดงสถานะ "Saved to Cloud"
4.  **สำรอง:** หากส่งไม่ผ่าน (เช่น เน็ตหลุดกระทันหัน) ระบบจะตัดเข้าสู่โหมดออฟไลน์ทันที

### สถานะ: 🔴 ออฟไลน์ (Offline)
1.  **ตรวจจับ:** แอปพลิเคชันตรวจพบว่าไม่มีการเชื่อมต่ออินเทอร์เน็ต
2.  **บันทึกท้องถิ่น:** ข้อมูลฟอร์ม, พิกัด GPS, และรูปภาพ (Base64) จะถูกบันทึกลงใน `localStorage` หรือ `IndexedDB` ของเบราว์เซอร์
3.  **แจ้งเตือน:** ระบบแจ้งผู้ใช้ว่า "บันทึกในเครื่องแล้ว (Waiting for Sync)"
4.  **รอจังหวะ (Queueing):** ข้อมูลจะเข้าคิวรอการอัปโหลด
5.  **ซิงค์อัตโนมัติ (Auto-Sync):** เมื่ออุปกรณ์กลับมาออนไลน์ (Reconnection), Service Worker จะทำการส่งข้อมูลในคิวขึ้น Server ทีละรายการจนครบ

## 3. ฟีเจอร์ที่ใช้งานได้ และ ไม่ได้ (Capabilities)

### ✅ ทำงานได้แม้ไม่มีเน็ต (Available Offline)
*   **การกรอกแบบฟอร์ม:** กรอกค่า Voltage, Ohm, และเลือกสถานะต่างๆ
*   **การถ่ายรูป:** ถ่ายรูปและเก็บรูปภาพไว้ในเครื่องชั่วคราว
*   **GPS:** ดึงพิกัดจากชิป GPS ในตัวเครื่อง (Device GPS ไม่ต้องใช้เน็ต)
*   **ลายเซ็นดิจิทัล:** เซ็นชื่อบนหน้าจอ
*   **การดูข้อมูลเก่า:** ดูรายการงานที่เคยโหลดมาแล้ว (Cached Data)

### ❌ ต้องการเน็ตเท่านั้น (Online Only)
*   **AI Vision Analysis:** การวิเคราะห์ภาพด้วย Gemini ต้องส่งรูปไปประมวลผลบน Cloud
*   **AI Voice Co-pilot:** การถอดเสียงภาษาไทยต้องใช้ API
*   **Real-time Maps:** การโหลดแผนที่ Google Maps (แต่พิกัด Lat/Lng ยังทำงานได้)

## 4. เทคนิคทางเทคนิค (Technical Mechanism)
*   **Service Workers:** สคริปต์เบื้องหลังที่ทำหน้าที่เป็น Proxy คอยดักจับ Network Request และจัดการ Cache
*   **Optimistic UI:** หน้าจอจะตอบสนองทันทีว่า "สำเร็จ" เมื่อผู้ใช้กดปุ่ม โดยไม่รอ Server ตอบกลับ เพื่อความลื่นไหลในการใช้งาน
*   **Conflict Resolution:** หากมีการแก้ไขข้อมูลเดียวกันจากหลายเครื่อง ระบบจะยึดตาม Timestamp ล่าสุด (Last Write Wins) หรือแจ้งเตือนให้ผู้ใช้ตรวจสอบ

## 5. ประโยชน์ที่ได้รับ (Benefits)
1.  **ความต่อเนื่อง (Continuity):** เจ้าหน้าที่สามารถลงพื้นที่ในจุดอับสัญญาณ (เช่น ภูเขา, พื้นที่ห่างไกล) ได้โดยงานไม่สะดุด
2.  **ความเร็ว (Speed):** แอปพลิเคชันตอบสนองไวมาก เพราะไม่ต้องรอโหลดข้อมูลจาก Server ทุกครั้ง
3.  **ความมั่นใจ (Integrity):** ลดความเสี่ยงข้อมูลสูญหายจากการที่เน็ตหลุดระหว่างกดบันทึก
